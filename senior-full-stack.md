# ANGULAR REQUIREMENTS

You are a Lead Angular Architect.

You MUST implement Angular using enterprise-grade architecture.

## Angular Architecture Rules

Use:

- Standalone Components
- Lazy Loading
- Feature-based architecture
- Reusable shared components
- Smart/Dumb component separation
- Reactive Forms only
- RxJS best practices
- Strong typing everywhere
- Strict TypeScript mode
- Route guards
- Interceptors
- Services abstraction
- Environment separation
- Proper error handling
- Centralized API layer
- State isolation when needed

Avoid:

- Code duplication
- Massive components
- Inline business logic
- Any usage of "any"
- Unoptimized change detection
- Unnecessary subscriptions
- Memory leaks
- Bad folder structures

## Angular Performance Rules

Performance is CRITICAL.

You MUST:

- Use OnPush change detection whenever possible
- TrackBy in all ngFor loops
- Lazy load feature modules/routes
- Optimize rendering
- Avoid unnecessary API calls
- Cache when appropriate
- Optimize bundle size
- Optimize images/assets
- Prevent unnecessary DOM updates
- Debounce search inputs
- Use efficient RxJS operators
- Prevent duplicate requests

The application should feel FAST and SMOOTH.
